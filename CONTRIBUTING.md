# Contribuyendo a galery_app

Gracias por tu interés en contribuir. Estas son las pautas para mantener el proyecto organizado.

## Reportar issues

- Usa el buscador para verificar si el issue ya fue reportado.
- Incluye pasos para reproducir, versión de Flutter/dispositivo y logs si aplica.

## Pull Requests

1. Crea un branch con nombre descriptivo: `fix/`, `feat/`, `refactor/`
2. Mantén los cambios enfocados en un solo propósito.
3. Ejecuta `flutter analyze` antes de enviar.
4. Actualiza el CHANGELOG.md si el cambio es relevante para usuarios.

## Estilo de código

- Sigue las convenciones estándar de Dart/Flutter.
- No agregues comentarios innecesarios.
- Usa nombres descriptivos para variables y funciones.
